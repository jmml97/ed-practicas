struct vs. class

  • Por defecto, todos los campos de struct son públicos, mientras que los de class son privados.
    Para el compilador, no existe diferencia entre ellos. De hecho, dentro de un struct puede haber métodos y constructores. Es decir, el siguiente bloque de código es equivalente a class Matriz.

struct Matriz {
  private:
    int filas, columnas;
};

  • class enfatiza la idea del encapsulamiento. Como norma general, usamos struct cuando todos los campos sean públicos.

Uso de class

Compilación separada

Archivo de cabecera (.h)

class Matriz {
    int fils, cols;
    double datos[50];
  public:
    int filas();
    int columnas();
    void set (int f, int c, double v);
    double get (int f, int c);
 
};

Archivo de definiciones (.cpp)

#include "matriz.h"
 
int Matriz::filas()
{
  return filas;
}
 
void Matriz::set (int f, int c, double v)
{
  datos[f*columnas+c] = v;
}

El puntero this

this es un puntero que apunta al objeto que hay en la llamada. Se crea automáticamente al mandar cualquier mensaje al objeto m.

cout << m.filas()
// Se ha creado un puntero this, como si fuera this = &m 
int Matriz::filas()
{
  return this->fils; // equivalente a (*this).fils 
}

Poner this->fils es completamente equivalente a poner solo fils, por lo que en general se usa esta última forma. Sin embargo, en casos como conflictos de nombres es útil:

int metodo (int filas)
{
  return this->filas * filas;
  // el primero se refiere al dato miembro, y el segundo al argumento 
}

Métodos ‘constantes’

Al poner const tras los paréntesis al definir un método, decimos que this es un puntero a un objeto constante.

class Matriz {
  // ... 
  public:
  int filas() const;  // El objeto no cambia (*this) 
  void set(int f, int c, double v);
  int get(int f, int c) const
}

Entonces, al definir this, el compilador tiene dos opciones:

  • const Matriz* this = &m;

  • Matriz* this = &m;

Constructores

La sintaxis para un constructor sería:

// En el archivo .h 
Matriz();
Matriz(int f, int c);
 
// En el archivo .cpp 
Matriz::Matriz()
{
  fils = cols = 0;
}
 
Matriz::Matriz(int f, int c)
{
  fils = f;
  cols = c;
  for (int i = 0; i < f*c; i++)
    datos[i] = 0.0;
}

Como vemos, podemos sobrecargar los constructores (y también los métodos), siguiendo la misma filosofía que con funciones.

Si no definimos ningún constructor, el compilador proporciona un constructor vacío, equivalente a:

Matriz() {}

Para construir vectores de Matrices, es necesario que exista un constructor sin parámetros:

Matriz m(2,3);  // ok 
Matriz v[10]; // falla si no existe constructor por defecto 

Llamada explícita a un constructor

Matriz m;
//... 
m = Matriz(2,3);

En m se machaca lo que hubiera, y se asigna un nuevo objeto Matriz de 2x3.

Cabecera de class Matriz

El archivo de cabecera quedaría:

class Matriz {
    double datos[50];
    int fils, cols;
  public:
    Matriz();
    Matriz(int f, int c);
    int filas() const {return fils;}
    int columnas() const; {return cols;}
    void set (int f, int c, double v);
    double get (int f, int c) const;
};

Si definimos el cuerpo de la función en el archivo .h, el compilador interpreta que el método es inline.
Para definir un método inline fuera de class, sólo podemos
hacerlo en el mismo archivo .h, tal y como lo haríamos en el .cpp, pero añadiendo el prefijo inline.

Sobre el estilo

Los datos miembro se pueden prefijar con m_ o simplemente con _.
Por ejemplo: double m_datos;

Precondiciones

Por ahora, puesto que no hemos visto las excepciones, gestionamos los posibles errores con precondiciones. Es deber del usuario garantizar que se cumplen las precondiciones, sabiendo que el comportamiento es indeterminado si no se cumplen.

assert

Sin embargo, hay situaciones donde se inclumplen las precondiciones sin darse cuenta. Para facilitar la depuración, tenemos la función assert, incluida en la cabecera cassert:

/**
  * @file matriz.cpp
  */
#include <cassert>
#include "matriz.h"
 
void Matriz::set(int f, int c, double v)
{
  assert (f >= 0 && f < fils && c >= 0 && c < cols);
  this->datos[f*cols+c] = v;
}

assert comprueba que se cumplen las condiciones entre paréntesis, y si no, explota el programa con un mensaje aclaratorio.

Cuando estemos seguros de que todo funciona, podemos compilar con

> g++ -DNDEBUG

que automáticamente elimina todas las ocurrencias de assert.

Objetos y memoria dinámica

int main()
{
Matriz *p;  // no declara ninguna matriz 
 
// reserva espacio en memoria dinámica para una matriz, y llama al constructor sin parámetros. 
p = new Matriz;
p = new Matriz(3,4);  // igual, pero llamando al constructor con dos parámetros 
 
// la liberamos como 
delete p;
 
// Se reserva un bloque de 100 matrices en memoria dinámica, y en cada una de ellas se llama al constructor por defecto. 
p = new Matriz[100];  // p apunta al inicio de un vector de 100 matrices 
 
// Ahora podríamos machacar las matrices: 
p[0] = Matriz(3,5);
cout << p->filas(); // imprime 3 
 
// Y liberarlas con 
delete[] p;
}

Memoria dinámica en campos privados

// matriz.h 
class Matriz {
    double *datos;
    int fils, cols;
  public:
    Matriz(int f, int c);
    int filas();
    int columnas();
    void set (int f, int c, double v);
    int get (int f, int c);
};
// matriz.cpp 
#include "matriz.h"
 
Matriz::Matriz (int f, int c)
{
  datos = new double[f*c];
  fils = f;
  cols = c;
}
// main 
#include "matriz.h"
 
int main()
{
Matriz m(2,3);  // reservada en memoria dinámica (un vector de 6 posiciones) 
m.set(0,0,3);
}

El resto de métodos son iguales que antes.

Asignación y copia

Al igual que ocurría con constantes a nivel de clase, la asignación entre objetos se rompe cuando usamos memoria dinámica.

int main()
{
  Matriz m1(2,3);
  Matriz m2(3,4);
 
  m1.set(0,0,3.5);
  m2.set(0,1,2.4);
 
  m1 = m2;
  /*
    La asignación funciona, pero no como esperamos.
    El puntero datos de m1 apunta ahora al mismo sitio
    que el puntero datos de m2. Por tanto, m1 y m2 comparten la misma posición en memoria,
    y cualquier cambio que hagamos luego en una se cambia en la otra.
    Además, el vector apuntado por m1 se pierde, no podemos hacerle delete.
  */
}

Por tanto, no podemos usar el operador de asignación tal cual para asignar matrices.

Tampoco funciona el paso por valor de objetos de tipo Matriz a funciones, e incluso la copa de matrices. Tampoco podemos liberar la memoria, por lo que necesitamos añadir nuevos métodos a nuestra clase Matriz.

Operador de asignación y constructor de copia. Destructores

En muchos casos necesitamos sobrecargar el operador de asignación, el constructor de copia o el destructor.

class Matriz {
    double *datos;
    int fils, cols;
  public:
    Matriz();
    Matriz(int f, int c);
    Matriz(const Matriz& m);  // copia 
    Matriz& operator=(const Matriz& m); // asignación 
    ~Matriz();  // Destructor 
    // ... 
    void Swap(Matriz& m);
};

Operador de asignación

Matriz& Matriz::operator=(const Matriz& m)
{
  Matriz aux(m);  // copiamos m 
 
  // Intercambiamos aux por *this 
  // (mejor meterlo en función swap) 
 
  // datos 
  double *p = aux.datos;
  aux.datos = this->datos;
  this->datos = p;
 
  // filas 
  int i = aux.fils;
  aux.fils = this->fils;
  this->fils = i;
 
  // columnas 
  i = aux.cols;
  aux.cols = this->cols;
  this->cols = i;
 
  return *this;
}

Al acabar la función, el compilador destruye aux(es un objeto local a la función). Como habíamos intercambiado aux por *this, se destruye el objeto *this original.

Matriz& Matriz::operator=(const Matriz& m)
{
  Matriz aux(m);
  swap(aux);
  return *this;
}

Observamos que también funcionaría la asignación m = m.

Constructor de copia

Matriz m1(2,3);
 
// ambas son llamadas al constructor de copia 
Matriz m(m1);
Matriz m=m1;

Destructor

 

Llamadas a constructores/destructores

El compilador garantiza que los objetos que usamos están siempre construídos. De igual forma, si el objeto deja de existir, es destruído automáticamente.

  • Podemos tener objetos globales

  • Variables static en funciones:

void f()
{
  static Matriz m(2,3);
  // existe en memoria estática 
}

La variable solo se crea la primera vez que se llama, y no se destruye al acabar la función, sino al término del programa. No existe fuera de la función, pero conserva el valor en llamadas sucesivas.

  • Conversión implícita

Si tenemos un constructor del tipo Matriz(int fc) que construye una matriz cuadrada, el compilador ya conoce cómo convertir implícitamente de un entero a una Matriz:

void f(const Matriz& m)
{
  cout << m.filas() << " " << m.columnas();
}
 
int main()
{
  f(3);
  // compila, y funciona tal y como se espera 
  // (con una matriz 3x3) 
}

Otro ejemplo, si tenemos la clase Complejo y tenemos un constructor que acepta un número real, a partir de entonces todas las operaciones con complejos funcionan también con reales (conversión implícita);

En cierto modo, un constructor con un parámetro es un conversor implícito. Si no queremos que se hagan conversiones implícitas, usamos la palabra clave explicit:

explicit Matriz(int fc);

  • Objetos como datos miembros

class Sistema {
  // no se construyen los objetos 
  Matriz izq,der;
public:
  Sistema(const Matriz& m1, const Matriz& m2) {
    // Justo antes de abrir la llave, se crean 
    // los campos izq,der (constructor por defecto) 
 
    // operador de asignación de la clase Matriz 
    izq = m1;
    der = m2;
  }
};

Si la clase Matriz no tuviera constructor por defecto, o si no queremos que lo use, usamos la lista de inicialización (ANTES de abrir llave), y especificamos cómo se construyen los objetos miembro de la clase.

NOTA: Como la clase Sistema no tiene memoria dinámica, no necesitamos constructor de copia, operador de asignación o destructor. No importa que la clase Matriz tenga internamente memoria dinámica, pues ya funciona correctamente según la definimos.

Función swap global

La función swap de dos matrices con el método tradicional sería muy ineficiente. Lo ideal sería poder hacer un swap global similar al método de la clase Matriz:

void swap(Matriz& m1, Matriz&m2)
{
  double *p = m1.dato;
  // ... 
}

Esta implementación sería la deseada, pero no compilaría: no podría acceder a la parte privada de m1 y m2.

Solución: declarar la función como “amiga” de la clase matriz:

class Matriz {
  //... 
  friend void swap(Matriz& m1, Matriz& m2);
  // ... 
};

Como vemos, es la clase Matriz la que tiene que “dar permiso” a otras funciones, clases, métodos de otras clases, etc. para acceder a su parte privada. Sin embargo, no es muy recomendable, pues se pierde el encapsulamiento. En este caso, sería mejor hacer algo así:

void swap(Matriz& m1, Matriz&m2)
{
  // llamada a un método público de Matriz 
  m1.swap(m2);
}

Creando nuestra clase Vector

  • Interfaz (.h)

class Vector {
    double *datos;
    int n;
    int reservados;
  public:
    Vector() : datos(0), n(0), reservados(0) {}
    Vector(int tam, double d = 0);
    ~Vector() {delete[] datos;}
    Vector(const Vector& v);
    Vector& operator= (const Vector& v);
    void set(int i, double d) {datos[i] = d;}
    double get(int i) const {return datos[i];}
    int size() const {return n;}
    void resize(int tam);
    void swap(Vector& v);
    void push_back(double d);
    void reserve(int tam);
    int capacity() const {return reservados;}
    void shrink_to_fit();  // C++11 
};

  • Implementación (.cpp)

// Constructor con parámetros 
Vector::Vector(int tam, double d) // aquí no se pone el valor por defecto 
{
  n = reservados = tam;
  datos = new double[tam];
 
  for (int i = 0; i < n; i++)
    datos[i] = d;
}
 
// Constructor de copia 
Vector::Vector(const Vector& v)
{
  /*
    Por cuestiones de eficiencia, no mantenemos
    los reservados originales, sino solo los usados.
  */
  datos = new double[v.n];
  n = v.n;
  reservados = v.n;
 
  for (int i = 0; i < n; i++)
    datos[i] = v.datos[i];
}
 
// Este método swap es muy eficiente 
void Vector::swap(Vector& v)
{
  // intercambiar punteros 
  double *p = datos;
  datos = v.datos;
  v.datos = p;
 
  // intercambiar tamaño 
  int i = n;
  n = v.n;
  v.n = i;
 
  // intercambiar reservados 
  i = reservados;
  reservados = v.reservados;
  v.reservados = i;
}
 
// Operador de asignación 
Vector& Vector::operator=(const Vector& v)
{
  /*
    Declaro un vector aux como copia de v, y lo
    intercambio con *this.
  */
  Vector aux(v);
  swap(aux);  // equivalente a this->swap(aux) y a aux.swap(*this) 
 
  return *this;
}
 
// petición para reservar, al menos, 'tam' posiciones nuevas 
void Vector::reserve(int tam)
{
  if (tam > reservados) {
    double *aux = new double[tam];
 
    for (int i = 0; i < n; i++)
      aux[i] = datos[i];
 
    delete[] datos;
    datos = aux;
    reservados = tam;
  }
}
 
// Añade un elemento al final 
void Vector::push_back(double d)
{
  /*
    Aprovechamos, y si no hay espacio,
    reservamos el doble de lo que había.
  */
  if (n == reservados) {
    if (n == 0)
      reserve(1); // porque 2*0 = 0 
    else
      reserve(2*n);
  }
  datos[n++] = d;
}
 
void Vector::resize(int tam)
{
  reserve(tam); // reserve ya comprueba si tam > reservados 
  n = tam;
}
 
// Se pone reservados al nivel de n, para no desperdiciar memoria 
void Vector::shrink_to_fit()
{
  Vector aux(*this);  // al copiar, se equipara n con reservados 
  swap(aux);
}

Observaciones:

  • No ponemos condiciones en la función set, para aumentar
    la rapidez.

  • Si hacemos Vector v(10), se construye un vector de 10 elementos,
    y se inicializa a 0 (parámetro por defecto).

Mejora en la clase Matriz: vector dinámico como dato miembro

class Matriz {
    Vector datos;
    int fils, cols;
  public:
    Matriz(int f, int c, double d = 0) : datos(f*c, d) {
      fils = f;
      cols = c;
      /*
        Si no usásemos la lista de inicialización, se construiría
        datos ANTES de abrir llave, con el constructor por defecto
        de la clase Vector (si no existiera, daría error de compilación).
        Entonces, tendríamos que hacer:
 
        datos.resize(f*c);
        for (int i = 0; i < f*c; i++)
          datos.set(i, d);
 
        O también, equivalentemente, una llamada explícita al constructor
        de la clase Vector:
 
        datos = Vector (f*c, d);
      */
    }
};

No necesitamos constructor de copia, destructor ni operador de asignación,
pues no usamos memoria dinámica explícitamente. El objeto datos de la clase Vector
resuelve sus problemas internamente, igual que, por ejemplo, un objeto string.
El destructor por defecto de la clase Matriz llama automáticamente al destructor
de la clase Vector.
